Mission Statement

To build sovereign digital infrastructure that a single independent engineer can own outright — architecture, data, and identity — rather than rent from platforms that can revoke access, sell the data, or disappear. Every service exists because an equivalent commercial product was judged inadequate, extractive, or untrustworthy: permit tracking not beholden to a vendor, terrain intelligence computed from physical law instead of licensed from a mapping monopoly, a browser that doesn't phone home, a community news outlet whose revenue stays in the community it serves.

The long-term purpose is broader than any one repository: a civic-technology substrate for a future where digital and physical infrastructure are the same infrastructure — where augmented and virtual reality are how people collaboratively build their own cities, and autonomous and robotic systems join that build-out under the same sovereign, auditable, provenance-tracked standards documented in this catalog.

Why It Exists

Independent verification, under duress, that a single person's engineering output can constitute a serious body of work — architected, tested, and documented to a standard that survives scrutiny, not a portfolio of toy projects.

How It's Built

Every repository is cross-referenced through a shared phase-tracking protocol, a common attribution and license standard, and a structural interconnection point (the XML intelligence pipeline) that every other service writes through.

What It Proves

Real, running, tested code: Rust unit test suites, live-provisioned cloud databases, TypeScript type-checked across two build targets, and a security/dependency posture audited and modernized on a fixed date, not a resume claim.

Where It's Going

From desktop dashboards to a mixed-reality, physically-integrated civic build-out — the architecture in this catalog is the substrate that transition is built on top of, not a rewrite that transition will require.

Novel Design Systems — Service Deliverables

Original engineering patterns designed and implemented within this estate, generalizable beyond it, and presented here as standalone service deliverables in their own right.

01
5D Cell Addressing Model MacroHarder
Extends the two-dimensional spreadsheet cell — ⟨column, row⟩ — into a five-coordinate address space: ⟨column, row, layer, tick, escape⟩. Layer gives volumetric (sheet-of-sheets) depth; tick and escape reserve temporal versioning and directional-write routing as first-class coordinates rather than bolted-on metadata.
02
Capability-Gated MCP Module Host MacroHarder
A module registry that validates every cross-service tool call against a declared manifest before dispatch, and enforces a read-only / read-write capability class per module — closing a real authorization gap where any tool name could otherwise be forwarded to any registered service.
03
SOV-RULES Taxonomy Guard SQLXML
A versioned XML taxonomy gate enforced immediately before every structural database write, so schema drift and malformed writes are rejected at the boundary rather than discovered downstream.
04
21-Day Cycle Governor + n+6 Stabilizer SQLXML
A self-resetting operational cycle (day-22 wipe logic) paired with a trailing 6-day recursive debugger that scopes recent history specifically to catch structural drift before it compounds — an audit standard for systems that must self-correct without human intervention.
05
Sovereign Provenance & Watermarking Suite RoadMaps
A four-part IP-forensics toolkit: BLAKE3 structural content fingerprinting, source-embedded canary tokens, DNS beacon call-home tokens for unauthorized-compile detection, and an AES-256-GCM authenticated append-only provenance ledger. A full security-standard specification, not a single script.
06
Node Certification Ledger (NCL) RoadMaps · SimCity
A three-tier append-only safety and seating ledger: Tier 1 generates and dispatches signed certification hashes, Tier 2 maintains the chain-linked public record and a discovery index, Tier 3 runs a variance-rate evaluation with an oscillation gate across fixed T-30/T-45 evaluation windows.
07
Albert Escaped Rays (AER) Protocol Estate-wide
A directional-write behavioral contract with a locale registry and an enforced one-read/two-write invariant, giving the estate a formal consistency guarantee for how data escapes from one service boundary into another.
08
3D Design Standard & Weighted Audit Scoring MacroHarder
A category-weighted PASS / WARN / FAIL compliance scorer for adjustable-UI design-token systems — the quality gate every dashboard preset in the estate is measured against before it ships.
09
Cross-Repo Phase Synchronization Protocol Estate-wide
A shared, machine-readable roadmap schema — one file per repository, aggregated into a single master roadmap — giving eleven independently deployed services one consistent source of truth for phase state, dependencies, and open work.
10
Autonomous IP-Infringement Detection Pipeline IP-Forensics
Zero-human-intervention hourly scanning across four public package ecosystems, combining verbatim-copy hashing, structural AST comparison, and unique-identifier detection to surface unauthorized derivative use as it happens, not months later.

Vision — The Physical-Digital Merger

Every system in this catalog is architected as an API-first, headless service precisely because the interface layer is not the destination. The desktop dashboard and the sovereign browser are the current interface; they are not the intended permanent one. The long-term plan is a transition to augmented and virtual reality as the primary way people see and collaboratively build their own civic infrastructure — walking the actual streets and seeing the actual permit pipeline, terrain model, and construction status overlaid in place, not read off a screen at a desk.

That transition requires the estate to include everyone in the build-out process, not just engineers with dashboard access — which is why every module in this system is designed around a public, capability-gated interface from day one. It also requires integrating autonomous and robotic systems into physical construction and maintenance under the same provenance, audit, and capability-gating standards already enforced in software — the same taxonomy guards, the same signed ledgers, extended to hardware. Solidarity Goggles is the working name for that mixed-reality presentation layer: a shared, not individually-siloed, augmented view of the build-out, designed for the people doing the building, not for a platform selling their attention.

None of this requires the current architecture to be rewritten. It requires exactly what has been built: sovereign data, capability-gated module boundaries, cryptographic provenance, and a phase-tracked roadmap — the substrate a mixed-reality and robotics layer plugs into, rather than the wall it has to be torn down to make room for.
